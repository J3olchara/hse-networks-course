## statistics/statistics.go

| Функция | Строки | Описание |
|---------|--------|----------|
| `CollectStatistics()` | 17-56 | **[ЗАДАЧА 3]** Сбор всех 8 показателей статистики за заданное время |
| `processPacket()` | 58-114 | Обработка каждого пакета и обновление счетчиков |
| `cleanupOldRequests()` | 116-122 | Очистка устаревших ARP запросов (старше 5 секунд) |
| `printStatistics()` | 124-163 | Форматированный вывод результатов статистики |

**Реализованные показатели:**

1. **Ethernet фреймы** (строка 64): Счетчик `TotalEthernetFrames++`
2. **ARP пакеты** (строка 77): Счетчик `TotalArpPackets++`
3. **Уникальные MAC** (строки 66-67): Map `UniqueMACAddresses`
4. **Широковещательные** (строки 69-71, 79-81): 
   - `BroadcastFrames` - все с dst MAC = ff:ff:ff:ff:ff:ff
   - `BroadcastArpFrames` - широковещательные + ARP
5. **Gratuitous ARP** (строки 83-85): Проверка через `utils.IsGratuitousArp()`
6. **Пары Request/Response** (строки 87-109): 
   - Сохранение Request с timestamp
   - Сопоставление с Response по IP адресам
   - Проверка временного интервала (< 5 секунд)
7. **Объем данных** (строки 73-75):
   - Фильтрация по MAC устройства и роутера
   - Суммирование размеров фреймов

## utils/parser.go

| Функция | Строки | Описание |
|---------|--------|----------|
| `IsGratuitousArp()` | 13-30 | Проверка, является ли ARP пакет Gratuitous ARP Request |
| `IsBroadcast()` | 32-35 | Проверка, является ли Ethernet фрейм широковещательным |
| `FormatArpPacket()` | 37-75 | Форматированный вывод ARP пакета с интерпретацией всех полей |
| `CalculateFrameSize()` | 77-84 | Расчет размера Ethernet фрейма с учетом минимального размера (64 байта) |
| `IsDeviceToRouter()` | 86-88 | Проверка, идет ли фрейм от устройства к роутеру |
| `IsRouterToDevice()` | 90-92 | Проверка, идет ли фрейм от роутера к устройству |

**Логика Gratuitous ARP (строки 14-28):**
- Operation = ARPRequest (1)
- Sender IP == Target IP
- Target MAC = 00:00:00:00:00:00 или ff:ff:ff:ff:ff:ff

**Расчет размера фрейма (строки 78-83):**
- Используется `packet.Metadata().CaptureLength`
- Минимальный размер Ethernet фрейма = 64 байта
- Учитывается padding

## Соответствие требованиям задания

### Задача 1: Захват всех пакетов ARP
- **Файл:** `capture/capture.go`
- **Функция:** `CaptureArpPackets()` (строки 13-56)
- **Особенности:** PROMISCUOUS режим, интерпретация всех полей ARP заголовка

### Задача 2: Выяснить MAC адрес роутера
- **Файл:** `sender/sender.go`
- **Функция:** `FindRouterMAC()` (строки 14-63)
- **Особенности:** Создание Ethernet фрейма, отправка ARP запроса, ожидание ответа

### Задача 3: Статистика (8 показателей)
- **Файл:** `statistics/statistics.go`
- **Функция:** `CollectStatistics()` (строки 17-56)
- **Обработка:** `processPacket()` (строки 58-114)
- **Особенности:** 
  - Показатели 1-2: строки 64, 77
  - Показатель 3: строки 66-67
  - Показатель 4: строки 69-71, 79-81
  - Показатель 5: строки 83-85
  - Показатель 6: строки 87-109
  - Показатели 7-8: строки 73-75

### Вспомогательные функции
- **Интерпретация ARP:** `utils/parser.go`, функция `FormatArpPacket()` (строки 37-75)
- **Определение Gratuitous ARP:** `utils/parser.go`, функция `IsGratuitousArp()` (строки 13-30)
- **Расчет размера фрейма:** `utils/parser.go`, функция `CalculateFrameSize()` (строки 77-84)

## Примечания

- Все функции документированы в исходном коде
- Код организован по пакетам согласно функциональности
- Каждая задача реализована в отдельном модуле
