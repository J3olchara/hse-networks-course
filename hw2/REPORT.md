## контрольные вопросы

### Для чего нужны ARP targeted requests/responses? Какова их структура?

ARP Targeted Request используется для:
- Определения MAC адреса устройства по известному IP адресу
- Отправляется когда узлу нужно отпарвить IP пакет, но в ARP кэше нет соответствующего MAC адреса
- На запрос отвечает устройство котому принадлежит запрашиваемый IP адрес

ARP Targeted Response используется для:
- Ответа на ARP Request с информацией о MAC адресе запрашиваемого IP
- Позволяет отправителю запроса обновить свой ARP кэш и установить соответсвие IP-MAC

Структура ARP пакета (28 байт):

| Поле | Размер | Значение для Request | Значение для Reply |
|------|--------|----------------------|---------------------|
| Hardware Type | 2 байта | 1 (Ethernet) | 1 (Ethernet) |
| Protocol Type | 2 байта | 0x0800 (IPv4) | 0x0800 (IPv4) |
| Hardware Address Length | 1 байт | 6 | 6 |
| Protocol Address Length | 1 байт | 4 | 4 |
| Operation | 2 байта | 1 (request) | 2 (reply) |
| Sender Hardware Address | 6 байт | MAC отправителя | MAC отвечающего |
| Sender Protocol Address | 4 байта | IP отправителя | IP отвечающего |
| Target Hardware Address | 6 байт | 00:00:00:00:00:00 | MAC получателя ответа |
| Target Protocol Address | 4 байта | IP цели | IP получателя ответа |

Отличие от Gratuitous ARP:
- В обычном ARP Request поля Sender IP и Target IP разные (запрашиваеться чужой IP)
- Target MAC в Request всегда 00:00:00:00:00:00, тк именно его мы и хотим узнать
- Reply отправляется unicast конкретному получателю, а не broadcast

### 8.2. Что такое Gratuitous ARP Request? Для чего используется? Какова структура?

Gratuitous ARP Request - это ARP запрос в котором отправитель запрашивает MAC адрес для своего собственого IP адреса (Sender IP == Target IP).

Назначение:
1. Объявление о присутсвии в сети и обновление ARP кэшей других устройств при подключении к сети
2. Обнаружение конфликта IP адресов (если другое устройство ответит на запрос своего же IP значит адрес занят)
3. Обновление ARP кэшей соседей при смене MAC адреса (напрмер при смене сетевой карты или failover)

Особености структуры:

| Поле | Значение | Отличие от обычного Request |
|------|----------|------------------------------|
| Operation | 1 (request) | Совпадает |
| Sender IP | IP устройства | Совпадает |
| Target IP | IP устройства | Совпадает с Sender IP |
| Target MAC | 00:00:00:00:00:00 или ff:ff:ff:ff:ff:ff | Нулевой или broadcast |
| Ethernet Destination | ff:ff:ff:ff:ff:ff | Broadcast |

### 8.3. Почему обнаружено N уникальных MAC адресов?

Количество обнаруженых уникальных MAC адресов зависит от количества активных устройств в локальной сети во время сбора статистики. При захвате трафика в promiscuous режиме учитываються все Ethernet фреймы.

Устройства в сети:
- Отправляют ARP запросы для обнавления своего кэша перед отправкой IP пакетов
- Отвечают на ARP запросы других устройств
- Отправляют Gratuitous ARP при подключении к сети или изменении конфигурации
- Периодически общаються с роутером для доступа в интернет

Каждое устройство имеет уникальный MAC адрес, поэтому кол-во обнаруженных MAC адресов примерно соответствует количеству активных устройств в сети.

### 8.4. Каким образом рассчитывается объем данных в байтах?

Метод расчета:

1. Фильтрация фреймов: учитываються только фреймы между устройством и роутером (проверка по Source MAC и Destination MAC)

2. Расчет размера каждого фрейма: используеться packet.Metadata().CaptureLength. Если размер меньше 64 байт используется минимальный размер Ethernet фрейма - 64 байта.

Структура Ethernet фрейма:

| Компонент | Размер | Учитывается libpcap? |
|-----------|--------|----------------------|
| Preamble | 7 байт | Нет |
| SFD | 1 байт | Нет |
| Destination MAC | 6 байт | Да |
| Source MAC | 6 байт | Да |
| EtherType | 2 байта | Да |
| Payload | 46-1500 байт | Да |
| Padding | 0-42 байта | Да |
| FCS | 4 байта | Зависит от настроек |

Минимальный размер фрейма с FCS: 64 байта. Для ARP пакетов: 14 байт заголовк + 28 байт payload + 18 байт padding + 4 байта FCS = 64 байта.

3. Суммирование: для каждого направления ведеться отдельный счетчик затем они суммируються.
